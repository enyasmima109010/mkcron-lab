# mkcron-lab

## 项目简介

mkcron-lab 是一个用于归档和发布多个独立 HTML 页面的仓库。这里存放的页面各自独立运行，不依赖特定后端服务，适合作为静态内容展示、工具页面或信息聚合页。

## 目录说明

```
mkcron-lab/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面所需的静态资源（图片、样式、脚本等）
├── README.md       # 本文件
└── LICENSE         # 项目许可证
```

- `pages/` 目录下每个子文件夹或文件对应一个独立页面，命名遵循功能或内容主题。
- `assets/` 目录按需组织，资源文件与页面分离管理。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，不绑定任何特定域名或网站。每个页面设计为可直接通过浏览器打开运行，或部署至任意静态托管服务。

页面内容可能涵盖工具、参考信息、演示页面等，具体用途见各页面内的注释或说明。由于页面独立维护，不同页面可能采用不同技术栈（纯 HTML/CSS/JS、Vue、React 等），但均以静态形式发布。

## 维护说明

- 页面更新以提交新文件或覆盖旧文件的方式进行，历史版本可通过 Git 记录追溯。
- 欢迎提交 Issue 或 Pull Request 来改进现有页面或增加新页面。提交时请遵循以下原则：
  - 页面内容不包含恶意代码、隐私收集或违反法律法规的信息。
  - 新增页面请在 `pages/` 下创建独立目录，并附带简要说明。
  - 资源文件请放入 `assets/` 对应子目录，避免与页面混放。
